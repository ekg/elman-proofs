## Standard Professional Format

### LaTeX/amsthm Standard (most math papers)

**Format**: **Theorem X.Y** (Optional Name). *Statement in italics.*

**Examples**:
- **Theorem 2.1** (Composition Depth Gap). *For a D-layer model processing sequences of length T: Linear temporal dynamics yields composition depth D. Nonlinear temporal dynamics yields composition depth D × T.*

- **Lemma 3.4** (Parity is Not Affine). *No affine function equals XOR on {0,1}².*

- **Theorem 5.2**. *Every bounded linear operator on a Hilbert space has an adjoint.* (no name, just number)

- **Definition 2.5** (Linear RNN). A linear RNN has dynamics h_t = Ah_{t-1} + Bx_t... (upright, not italic)

### Key Conventions from Published Math Papers

1. **Number comes first**: "Theorem 2.1" not "Theorem (Gap 2.1)"
2. **Name in parentheses after number**: "(Name)" is optional and follows the number
3. **Period after heading**: "Theorem 2.1." or "Theorem 2.1 (Name)."
4. **Statement is italic** for theorems/lemmas/propositions
5. **Statement is upright** for definitions/examples/remarks
6. **Numbering is hierarchical**: Section.Theorem (2.1, 2.2, 3.1, etc.)

### Examples from arXiv Papers

**From typical algebraic geometry paper**:
```
Theorem 4.3 (Riemann-Roch). Let X be a smooth projective curve...
```

**From typical complexity theory paper**:
```
Theorem 2.1. PARITY ∈ TC⁰ \ AC⁰.

Proof. [proof here] □
```

**From typical functional analysis paper**:
```
Lemma 3.2 (Banach Fixed Point). Let (X,d) be a complete metric space...
```

## What the ctheorems Package Should Produce

The `ctheorems` package in `traditional-math-style.typ` is configured to produce:

```typst
#theorem(title: "Composition Depth Gap")[body]
```

Should render as:
```
Theorem 2.1 (Composition Depth Gap). [body in italics]
```

## The Bug in Current Usage

Looking at `02-foundations.typ`, theorems are called as:
```typst
#theorem("Composition Depth Gap")[
  For a D-layer model...
]
```

But the template has:
```typst
titlefmt: title => text(weight: "bold")[(#title)]
```

This wraps the ENTIRE title (including auto-number) in parentheses, producing:
```
(Theorem 0.1 Composition Depth Gap): ...
```

## Solution

### Option 1: Fix the titlefmt in traditional-math-style.typ

Change from:
```typst
titlefmt: title => text(weight: "bold")[(#title)]
```

To:
```typst
titlefmt: title => [#text(weight: "bold")[#title].]
```

And let ctheorems handle the numbering properly.

### Option 2: Use ctheorems properly with hierarchical numbering

The theorems should be numbered by section:
- Section 2 theorems: 2.1, 2.2, 2.3
- Section 3 theorems: 3.1, 3.2, 3.3

Not manually numbered 0.1, 0.2, 0.3 (which makes no sense).

### Option 3: Remove parentheses entirely

Change:
```typst
titlefmt: title => text(weight: "bold")[(#title)]
```

To just:
```typst
titlefmt: title => text(weight: "bold")[#title]
```

So it produces:
```
Theorem 2.1 (Composition Depth Gap). Statement...
```

Not:
```
(Theorem 2.1 (Composition Depth Gap)): Statement...
```

## What NOT to Do

❌ `(Composition Depth Gap 0.1): Statement`
❌ `Theorem (Composition Depth Gap 0.1). Statement`
❌ `0.1 (Composition Depth Gap): Statement`
❌ `[Theorem] Composition Depth Gap (0.1)`

## Implementation Fix

The file `traditional-math-style.typ` lines 62-64 should be:

```typst
#let theorem-base = thmbox(
  "theorem",
  "Theorem",
  fill: none,
  stroke: none,
  inset: 0pt,
  radius: 0pt,
  padding: (top: 1.2em, bottom: 1.2em),
  namefmt: name => [(#name)],  // Name in parens
  titlefmt: title => [#strong[#title].],  // Bold with period
  bodyfmt: body => emph[#body]  // Italicized
)
```

The key is:
- `namefmt` adds parentheses around the NAME only
- `titlefmt` makes the whole heading (Theorem X.Y (Name)) bold with period
- Let ctheorems auto-number properly
